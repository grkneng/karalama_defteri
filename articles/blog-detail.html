JavaScript Event Loop Derinlemesine İnceleme

javascript performance web
Author ID · 12 yorum · 15 Ocak 2026

JavaScript tek iş parçacıklı (single-threaded) bir çalışma modeline sahiptir. Bu, aynı anda yalnızca bir işlem yürütebildiği anlamına gelir. Peki tarayıcı nasıl olur da aynı anda hem animasyonları, hem kullanıcı etkileşimlerini hem de ağ isteklerini yönetebilir?

Cevap: Event Loop. Event Loop; Call Stack, Callback Queue ve Microtask Queue arasında koordinasyon sağlayan bir zamanlayıcıdır.

Call Stack

Call Stack, çalışan fonksiyonların tutulduğu veri yapısıdır. Fonksiyon çağrılır, stack’e eklenir. İşini bitirir, çıkar. Basit. Deterministik. Mekanik.

Callback Queue

setTimeout gibi API’ler doğrudan stack’e girmez. Önce Web API tarafında bekler, sonra queue’ya düşer. Event Loop, stack boşsa bu callback’i stack’e iter.

Microtask Queue

Promise’ler burada çalışır. Microtask’ler, normal callback’lerden önce işlenir. Bu yüzden Promise zincirleri beklenenden daha “öncelikli” davranır.

Event Loop bir sihir değildir. O bir sıraya koyma algoritmasıdır.

Performans sorunları çoğu zaman bu kuyruğun yanlış anlaşılmasından doğar. Uzun süren senkron işlemler UI’yi bloke eder. Bu yüzden CPU-bound işlemleri Web Worker’a taşımak mantıklıdır.
