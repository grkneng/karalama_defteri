Veri Yapısı ve Entegrasyon

MicroFlow akışına içerik sağlamak için gereken JSON şeması ve medya yönetim kuralları.

Uygulama, sunucu tarafında oluşturulan statik bir data.json dosyasından beslenir. Bu yapı, sonsuz kaydırma (infinite scroll) ve lazy-loading özelliklerinin performanslı çalışmasını sağlar. Aşağıdaki şema, her bir gönderi kartının temel yapı taşlarını tanımlar.

JSON Şeması

Her nesne benzersiz bir id içermelidir. Medya türü, render motorunun hangi bileşeni (Carousel, Video Player veya Image) yükleyeceğini belirler. 

{
  "id": 42,
  "author": "Yazılım Günlükleri",
  "email": "dev@microflow.io",
  "avatar": "https://cdn.example.com/u/42.jpg",
  "title": "Asenkron Veri Akışı",
  "text": "Fetch API kullanırken dikkat edilmesi gerekenler...",
  "media_type": "gallery", 
  "media_url": [
    "https://cdn.example.com/img1.jpg",
    "https://cdn.example.com/img2.jpg"
  ],
  "meta_text": "API Docs · 5 dk okuma"
}

Yukarıdaki örnekte media_type değeri "gallery" olduğu için, sistem otomatik olarak Bootstrap Carousel bileşenini başlatacaktır. Video içerikler için embed linkleri doğrudan kullanılabilir.

Dikkat Edilmesi Gerekenler

Veri setinizdeki görsellerin en-boy oranları, kart tasarımının bütünlüğü için kritiktir. Önerilen oran 16:9 veya 4:3'tür. Ayrıca avatar görselleri otomatik olarak dairesel (rounded-circle) formata dönüştürülür.

Kaynakça
  • JSON Data Interchange Format - ECMA-404 Standardı.
  • Bootstrap 5 Components - Kart ve Carousel yapıları hakkında detaylı bilgi.
  • MDN Web Docs, "Lazy Loading Strategies for Images".
